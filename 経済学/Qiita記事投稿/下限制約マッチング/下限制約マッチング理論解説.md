> "Strategyproof matching with regional minimum and maximum quotas."  
> *Artificial Intelligence*, 235, 40–57.

## 1. 概要

### 1.1 マッチング理論の歴史

マッチング理論の出発点は Gale & Shapley (1962) の古典的論文にある。彼らは「大学入試問題」と「結婚問題」を定式化し、**受入保留アルゴリズム（Deferred Acceptance Algorithm; DA）** を提案した。DA は次の 2 つの性質を同時に満たすことが証明されている。

- **安定性（Stability）**: いかなる学生・学校のペアも、互いに現在の相手より相手を好む「ブロッキングペア」が存在しない
- **耐戦略性（Strategyproofness）**: 学生が自分の選好を偽って申告しても、利得を得られない

これ以降、マッチング理論は経済学・コンピュータ科学の両分野で急速に発展し、研修医の病院配属（米国 NRMP、日本の医師臨床研修マッチング）、学校選択問題、腎臓交換プログラムなどの実社会への応用が次々に実現された。

### 1.2 従来研究の限界

標準的な DA は「各学校に個別最高定員 $q_c$ がある」という設定を前提としている。しかし現実の配属問題には、これだけでは捉えられない制約が多数存在する。

**課題となる制約の例:**

| 制約の種類   | 実例                                                                 |
| ------------ | -------------------------------------------------------------------- |
| 個別最低定員 | 授業成立のために必要な最低履修者数、病院の診療科存続に必要な研修医数 |
| 地域最高定員 | 都市部への医師集中を避けるための上限（日本の医師偏在対策）           |
| 地域最低定員 | 離島・へき地病院への研修医配属義務、軍の兵科ごとの充足要件           |

このような制約下では、安定マッチングが存在しないことが知られている。すなわち、**公平性（fairness）と非浪費性（nonwastefulness）は両立しない**。

### 1.3 本論文の新規性

Goto et al. (2016) は以下の 3 点で従来研究を大きく前進させた。

**① 計算複雑性の解明**  
地域の構造に制限がない一般の場合、「実行可能なマッチングが存在するか」という判定問題が **NP 完全**であることを証明した（Theorem 1, 2）。一方、地域が**階層木**（hierarchical tree）をなす場合は**線形時間**で判定できる（Theorem 3）。

**② 階層的地域制約に対する 2 つの耐戦略性メカニズム**  
個別・地域の最低定員と最高定員が同時に存在する最も一般的な設定に対して、耐戦略性かつ公平なメカニズムを 2 つ開発した。

- **PLDA-RQ**: 優先リスト順にコントラクトを貪欲に採用する
- **RSDA-RQ**: ラウンドロビン順で各学校が学生を選ぶ

**③ 弱められた非浪費性の導入と保証**  
安定マッチングが存在しない状況で、それぞれのメカニズムが満たす**弱い非浪費性**（PL-nonwastefulness, school-equitable-nonwastefulness）を定義し、両メカニズムがこれを達成することを証明した。

## 2. 前提知識

### 2.1 市場のモデル

市場は以下のタプルで定義される。

$$\mathcal{M} = (S, C, R, p, q, \succ_S, \succ_C, X, \succ_{PL})$$

| 記号                       | 意味                                                                         |
| -------------------------- | ---------------------------------------------------------------------------- |
| $S = \{s_1, \ldots, s_n\}$ | 学生の集合                                                                   |
| $C = \{c_1, \ldots, c_m\}$ | 学校の集合                                                                   |
| $R$                        | 地域の集合。各地域 $r \subseteq C$ は学校の部分集合                          |
| $p_r,\ q_r$                | 地域 $r$ の最低定員・最高定員（$0 \leq p_r \leq q_r$）                       |
| $\succ_s$                  | 学生 $s$ の学校に対する厳密選好順序                                          |
| $\succ_c$                  | 学校 $c$ の学生に対する厳密優先順序                                          |
| $X = S \times C$           | 全コントラクトの集合（$(s, c)$ は「$s$ が $c$ に割り当てられる」を意味する） |
| $\succ_{PL}$               | 優先リスト（コントラクト間の全順序）                                         |

**仮定:** すべての学生はすべての学校に許容可能で、逆も成り立つ。これにより、最低定員が小さく学生数が十分であれば、必ず実行可能マッチングが存在する。

### 2.2 実行可能性

マッチング $X' \subseteq X$ の実行可能性は以下の 2 条件で定義される。

**Student-feasible（学生実行可能）:**
$$\forall s \in S,\quad |X'_s| = 1$$
各学生がちょうど 1 つの学校に割り当てられている。

**School-feasible（学校実行可能）:**
$$\forall r \in R,\quad p_r \leq |X'_r| \leq q_r$$
各地域の学校に割り当てられた学生数が定員範囲内に収まっている。

両方を同時に満たすとき、$X'$ は**実行可能**と呼ぶ。

### 2.3 コントラクト理論の枠組み

本論文は Hatfield & Milgrom (2005) の**コントラクトによるマッチング（Matching with Contracts）** の枠組みを採用している。コントラクト $(s, c) \in X$ は「学生 $s$ と学校 $c$ のマッチング」を表す抽象的な取引単位であり、これにより複雑な制約を選択関数（choice function）として統一的に表現できる。学校側の選択関数 $Ch_C: 2^X \to 2^X$ が以下の 3 条件を満たすとき、一般化 DA はStrategyproofになる（Hatfield & Milgrom, 2005）。

| 条件                                                                | 意味                                                                         |
| ------------------------------------------------------------------- | ---------------------------------------------------------------------------- |
| **代替可能性（Substitutability）**                                  | コントラクト $x$ が $X'$ で却下されるなら、$X' \subseteq X''$ でも却下される |
| **集約需要の法則（Law of Aggregate Demand）**                       | コントラクト集合が大きくなると、採用数は単調非減少                           |
| **却下コントラクトの無関係性（Irrelevance of Rejected Contracts）** | 採用されないコントラクトを追加しても、他の採用結果に影響しない               |

本論文ではこれらを一括して保証する**単一改善性（Single-Improvement Property）**（Definition 9）を導入し、各メカニズムの選択関数がこの性質を満たすことを示している。

### 2.4 公平性と非浪費性

**公平性（Fairness）— Definition 1:**  
学生 $s$ が $(s, c) \in X'$ に割り当てられているとき、$c' \succ_s c$ かつ $s \succ_{c'} s'$ を満たす $(s', c') \in X'$ が存在するなら、$s$ は $s'$ に対して**正当な羨望（justified envy）**を持つ。正当な羨望が存在しないとき、マッチングは**公平**である。

直感的に言えば、「自分より優先度が低い学生が、自分の方が好きな学校に通っている」状況がないことを意味する。

**非浪費性（Nonwastefulness）— Definition 2:**  
学生 $s$ がより好む学校 $c'$ に移動でき（$c' \succ_s c$）、かつその移動が実行可能なマッチングを維持できるなら、$s$ は $c'$ の空席を**主張**できる。誰も空席を主張できないとき、マッチングは**非浪費的**である。

**安定性（Stability）= 公平性 ∧ 非浪費性**

最低定員が存在すると、公平性と非浪費性は一般に**両立しない**（Example 1）。人気のない学校に学生を押しつけなければならない場面では、その学生に対する公平性違反か、人気校の空席の無駄遣いかのどちらかを選ばざるを得ない。

### 2.5 弱められた非浪費性

安定マッチングが存在しない状況で、それぞれのメカニズムが保証する弱い非浪費性を導入する。

> **注: PL（Priority List / 優先リスト）とは**  
> PL は市場全体で統一されたコントラクトの優先順序 $\succ_{PL}$ を指す。各コントラクト $(s, c)$ は「学校 $c$ における学生 $s$ の優先度 $\text{rank}_c(s)$」を主キー、学校のタイブレーク順を副キーとして全順序が定まる（Section 2.1 の $\succ_{PL}$ 参照）。  
> 「PL-」を冠した性質はすべて、この全体的な優先順序を基準として通常の定義を弱めたものである。すなわち、「優先リスト上で現在の割当より上位のコントラクトを通じてのみ空席主張・羨望を認める」という制限を課すことで、定員制約下でも成立しやすくしている。

**PL-非浪費性（Definition 3）:**  
学生 $s$ の空席主張が正当と認められるのは、優先リストがその主張を支持する場合のみ、すなわち $(s, c') \succ_{PL} (s, c)$ の場合のみ。

**学校公平的非浪費性（Definition 4）:**  
学生 $s$ を $c$ から $c'$ へ移動させることが $c$ と $c'$ の学生数のアンバランスを**真に縮小する**（$|X'_{c'}| - |X'_c| \geq 2$）場合のみ、主張を正当と認める。

**PL 公平性（Definition 5）:**  
通常の公平性の強化版。優先リストに基づいた正当な羨望がないことを要求する。

### 2.6 地域の階層木構造

**階層的地域（Definition 6）:**  
地域集合 $R$ が hierarchical とは、任意の $r, r' \in R$ について $r \cap r' = \emptyset$、$r \subset r'$、$r' \subset r$ のいずれかが成立すること。

階層性が成り立つとき、地域集合は木で表現できる（Definition 7）。

```
根 (全学校 C)
├── 地域 r₁ (学校グループA)
│   ├── 葉 {c₁}
│   └── 葉 {c₂}
└── 地域 r₂ (学校グループB)
    ├── 葉 {c₃}
    └── 葉 {c₄}
```

この階層木において、「隣接するノード間の定員の整合性条件」が成立することを仮定する。

$$\sum_{r' \in \text{children}(r)} p_{r'} \leq p_r \leq q_r \leq \sum_{r' \in \text{children}(r)} q_{r'}$$

この条件は、子ノードの最低定員の合計が親の最低定員を超えず、かつ子の最高定員の合計が親の最高定員を下回らないことを意味し、定員体系の論理的整合性を保証する。

### 2.7 期待最小カウント

DA の中で「現在のコントラクト集合を拡張できるか」を効率的に判定するための概念。

**期待最小カウント（Expected Minimum Count） — Definition 10:**

$$e_r(X') :=
\begin{cases}
|X'_r| & \text{if } |r| = 1 \quad \text{（葉ノード）} \\
\displaystyle\sum_{r' \in \text{children}(r)} \max\bigl(e_{r'}(X'),\ p_{r'}\bigr) & \text{otherwise}
\end{cases}$$

$e_r(X')$ は「現在のマッチング $X'$ に対して、地域 $r$ の全下位地域が最低定員を満たすために**必要な最小学生数**」を表す。

**Theorem 4（半実行可能性の特徴付け）:**

$$X' \text{ が semi-school-feasible} \iff \forall r \in R,\quad e_r(X') \leq q_r$$

「semi-school-feasible」とは、$X'$ を拡張することで school-feasible なマッチングが得られる、という意味である。DA の各ステップでこの条件を確認することで、将来的な実行可能性を保ちながら貪欲にコントラクトを採用できる。

**計算例（Fig. 1 の木）:**  
$X' = \{(s_1, c_1), (s_2, c_1), (s_3, c_1), (s_4, c_1)\}$ のとき、$c_2$ の最低定員が 2 であれば:

$$e_{\{c_1, c_2\}}(X') = |X'_{c_1}| + \max(|X'_{c_2}|, p_{c_2}) = 4 + \max(0, 2) = 6 > 5 = q_{\{c_1, c_2\}}$$

よってこの $X'$ は semi-school-feasible でない（$c_1$ に詰め込みすぎて $c_2$ の最低定員を後から満たせない）。

## 3. アルゴリズム

### 3.1 一般化 DA（Mechanism 1）

すべての DA ベースメカニズムの骨格。学校側の選択関数 $Ch_C$ の定義だけが異なる。

```python
def generalized_da(market: Market, ch_c_func) -> frozenset[tuple[int, int]]:
    """一般化 DA（Mechanism 1）の骨格。ch_c_func だけがメカニズム間で異なる。"""
    rejected: set[tuple[int, int]] = set()

    while True:
        # Ch_S: 各学生が却下されていないコントラクトの中から最善の学校を選ぶ
        X_prime: set[tuple[int, int]] = set()
        for s in range(market.n_students):
            for c in market.student_prefs[s]:          # 選好順（好きな順）
                if (s, c) not in rejected:
                    X_prime.add((s, c))
                    break

        X_prime_fs = frozenset(X_prime)

        # Ch_C: 学校側の選択関数（PLDA-RQ / RSDA-RQ で実装が異なる）
        X_double_prime = ch_c_func(X_prime_fs, market)

        if X_prime_fs == X_double_prime:
            return X_double_prime               # 収束 → マッチング確定

        rejected |= X_prime_fs - X_double_prime  # 棄却されたコントラクトを記録
```

**収束保証:** Re は単調増加し $|X|$ で有界なので、有限回で終了する。

### 3.2 PLDA-RQ（Priority List based DA with Regional Quotas）

#### 選択関数 $Ch_C$（Definition 11）

```python
def ch_c_plda(
    X_prime: frozenset[tuple[int, int]],
    market: Market,
    priority_list: list[tuple[int, int]],   # 全コントラクトを PL 順に並べたリスト
) -> frozenset[tuple[int, int]]:
    """PLDA-RQ の学校側選択関数 Ch_C（Definition 11）。"""
    Y: frozenset[tuple[int, int]] = frozenset()

    for (s, c) in priority_list:            # PL 順（上位から）
        if (s, c) not in X_prime:
            continue                         # 今回の提案に含まれないコントラクトはスキップ

        Y_candidate = Y | {(s, c)}
        e_vals = expected_min_count_all(Y_candidate, market.region_tree)

        # 採用しても全地域の最高定員 q_r を超えないか確認（Theorem 4 の条件）
        if all(e_vals[node] <= node.max_quota
               for node in market.region_tree.all_nodes()):
            Y = Y_candidate                  # 採用
        # else: 棄却（Y には追加しない）

    return Y
```

**直感:** 優先リストの上位から順に「今これを採用しても将来の最低定員が詰まらないか」を確認し、問題なければ採用する。期待最小カウントを閾値として使うことで、将来の実行可能性を局所的に保証する。

#### 動作例（Example 3: Fig. 1 の市場）

学生 $s_1, \ldots, s_8$ の第 1 志望はほぼ全員 $c_4$ だが、$c_4$ の定員は 3。

**ラウンド 1:** 全員が $c_4$ か $c_1$ を選択。PL 順に採用すると $c_4$ は定員 3 で打ち止め（$s_6, s_5, s_4$ 採用、$s_3, s_2, s_1$ 棄却）。

**ラウンド 2:** $s_1, s_2, s_3$ が第 2 志望 $c_3$ へ。しかし $c_3, c_4$ の地域定員 5 に対して $e = 6$ になるため $s_4$ の $c_4$ 志望が棄却される。

**ラウンド 3:** $s_4$ が $c_1$ へ。ただし全体定員 8 に対して $e = 9$ になるため $s_8$ の $c_1$ 志望が棄却。

**ラウンド 4:** $s_8$ が $c_2$ へ。全定員を満たす実行可能マッチングが得られる。

**最終結果:**

| 学生            | 割当学校 | 志望順位  |
| --------------- | -------- | --------- |
| $s_1, s_2, s_3$ | $c_3$    | 2位       |
| $s_4, s_7$      | $c_1$    | 2位 / 1位 |
| $s_5, s_6$      | $c_4$    | 1位       |
| $s_8$           | $c_2$    | 2位       |

#### 保証される性質（Theorems 5–9）

| 性質                           | 成立                    |
| ------------------------------ | ----------------------- |
| 耐戦略性                       | ✓（Theorem 5）          |
| 公平性                         | ✓（Theorem 6）          |
| PL-非浪費性                    | ✓（Theorem 7）          |
| PL-公平性                      | ✓（Theorem 8）          |
| 学生最適 PL-公平かつ PL-非浪費 | ✓（Theorem 9）          |
| 非浪費性（通常）               | ✗（公平性との両立不可） |

**Theorem 9（最適性）:** PLDA-RQ は「PL-公平かつ PL-非浪費」を満たす全マッチングの中で、全学生に最も好まれるマッチング（**学生最適マッチング**）を実現する。

### 3.3 RSDA-RQ（Round-robin Selection DA with Regional Quotas）

#### 選択関数 $Ch_C$（Definition 12）

```python
def ch_c_rsda(
    X_prime: frozenset[tuple[int, int]],
    market: Market,
    round_robin_order: list[int],           # 学校インデックスのラウンドロビン順
) -> frozenset[tuple[int, int]]:
    """RSDA-RQ の学校側選択関数 Ch_C（Definition 12）。"""
    Y: frozenset[tuple[int, int]] = frozenset()
    rejected: set[tuple[int, int]] = set()

    while True:
        # 未決定のコントラクト（採用も棄却もされていない）
        undecided = {(s, c) for (s, c) in X_prime
                     if (s, c) not in Y and (s, c) not in rejected}
        if not undecided:
            break

        progress = False
        for c in round_robin_order:         # ラウンドロビン順に学校を選ぶ
            # 学校 c の未決定コントラクトを優先度順（高い順）に取得
            candidates = sorted(
                [(s, c2) for (s, c2) in undecided if c2 == c],
                key=lambda sc: market.school_rank(sc[1], sc[0])  # rank が小さい = 高優先
            )
            if not candidates:
                continue

            s, c_chosen = candidates[0]     # 最高優先度の学生を選ぶ
            Y_candidate = Y | {(s, c_chosen)}
            e_vals = expected_min_count_all(Y_candidate, market.region_tree)

            if all(e_vals[node] <= node.max_quota
                   for node in market.region_tree.all_nodes()):
                Y = Y_candidate              # 採用
            else:
                rejected.add((s, c_chosen)) # 棄却

            progress = True

        if not progress:
            break

    return Y
```

**直感:** 各学校が「自分のターン」に 1 人ずつ最高優先度の学生を選ぶ。ラウンドロビンによって学校間で選択機会が均等に与えられ、特定学校への学生集中が緩和される。

**PLDA-RQ との本質的な違い:** PLDA-RQ は優先リストのグローバルな順序を基準とするが、RSDA-RQ は「学校ごとの優先度」と「選択順番（ラウンドロビン）」を基準とする。この違いにより、RSDA-RQ は学校間の学生数バランスを重視した結果を生む。

#### 動作例（Example 4: 同一市場）

ラウンドロビン順: $c_1 \to c_2 \to c_3 \to c_4 \to c_1 \to \ldots$

ラウンド 2 では、ラウンドロビン順で $c_1, c_3, c_4, c_1, c_3, c_4, c_3$ と選んでいき、$c_4$ の定員が先に埋まる。

**最終結果:**

| 学生  | PLDA-RQ      | RSDA-RQ             |
| ----- | ------------ | ------------------- |
| $s_1$ | $c_3$（2位） | $c_3$（2位）        |
| $s_2$ | $c_3$（2位） | $c_3$（2位）        |
| $s_3$ | $c_3$（2位） | $c_2$（3位） ← 差異 |
| $s_4$ | $c_1$（2位） | $c_1$（2位）        |
| $s_5$ | $c_4$（1位） | $c_4$（1位）        |
| $s_6$ | $c_4$（1位） | $c_4$（1位）        |
| $s_7$ | $c_1$（1位） | $c_1$（1位）        |
| $s_8$ | $c_2$（2位） | $c_1$（1位） ← 差異 |

#### 保証される性質（Theorems 10–12）

| 性質               | 成立                                      |
| ------------------ | ----------------------------------------- |
| 耐戦略性           | ✓（Theorem 10）                           |
| 公平性             | ✓（Theorem 11）                           |
| 学校公平的非浪費性 | ✓（Theorem 12）                           |
| PL-非浪費性        | ✗                                         |
| 学生最適保証       | ✗（最適マッチングが存在しない場合もある） |

**注意:** 公平性と学校公平的非浪費性は HM 安定性を含意しないため、RSDA-RQ は学生最適マッチングを保証しない（Theorem 12 の証明後の注記）。

### 3.4 実行可能マッチングの存在判定（Theorem 3）

階層木が与えられたとき、以下の線形時間アルゴリズムで実行可能マッチングの存在を判定できる。

```python
def check_feasible_matching_exists(tree: RegionTree) -> bool:
    """
    実行可能マッチングの存在判定（Theorem 3）。線形時間。
    元のノードの定員値は変更せず、修正後の値を別辞書で管理する。
    """
    revised_p: dict[RegionNode, int] = {}
    revised_q: dict[RegionNode, int] = {}

    def _revise(node: RegionNode) -> None:
        if node.is_leaf():
            # 葉ノード: 元の定員をそのまま使用
            revised_p[node] = node.min_quota
            revised_q[node] = node.max_quota
        else:
            # 子を先に処理（ボトムアップ）
            for child in node.children:
                _revise(child)

            sum_p = sum(revised_p[child] for child in node.children)
            sum_q = sum(revised_q[child] for child in node.children)

            # 親の制約と子の合計の整合をとる
            # p_r: 子の合計が親の下限を上回れば子の合計が実効的な下限
            revised_p[node] = max(node.min_quota, sum_p)
            # q_r: 子の合計が親の上限を下回れば子の合計が実効的な上限
            revised_q[node] = min(node.max_quota, sum_q)

    _revise(tree.root)

    # 全ノードで修正後の p_r ≤ q_r なら実行可能マッチングが存在
    return all(revised_p[node] <= revised_q[node]
               for node in tree.all_nodes())
```

**正当性の直感:** 子の最低定員の合計が親の最低定員より大きければ、親の制約は実質的に非拘束（子の合計が実効的な下限になる）。逆に子の最高定員の合計が親の最高定員より小さければ、親の制約も非拘束。修正後に $p_r > q_r$ となるノードがあれば、その地域の定員体系が矛盾しており、実行可能マッチングは存在しない。

### 4.2 PLDA-RQ vs RSDA-RQ

#### メリット・デメリット比較

| 観点                   | PLDA-RQ                                         | RSDA-RQ                                             |

| **学生厚生**           | ◎ 高い（実験全条件で RSDA-RQ より平均順位が良い） | △ やや低い（ラウンドロビンが学生選好と無関係なため） |

| **政策適性**           | 学生選好を最大限尊重したい場面                   | 地域・学校間の均等配分が重要な場面                  |

**選択基準:**
- 学生の希望を最大限尊重したい → **PLDA-RQ**
- 各学校・地域への均等配分を重視したい → **RSDA-RQ**

**補足 — ラウンドロビン順の設計について:**  
RSDA-RQ はラウンドロビン順の選択がメカニズムの設計者に委ねられており、その順序が配分結果に直接影響する。例えば人気校を先頭に置くと先に選択機会を得た人気校に優秀な学生が集まりやすくなる。この点では PLDA-RQ の方が設計上の恣意性が低い。

### 4.3 PLDA-RQ / RSDA-RQ vs ACDA（人工キャップ）

ACDA は地域定員を個別定員に変換して近似するため、本質的に柔軟性を失う。

**理論的な問題点:**
- 地域内でどの学校に制約を割り当てるかの選択が恣意的
- 個別定員への変換後、地域内での学生配分の自由度が大幅に低下
- 結果として **wastefulness が高くなる**（claiming students が多い）

**実験結果の要約（binary tree, pc = 0）:**

| alpha | PLDA-RQ claiming率 | RSDA-RQ claiming率 | ACDA claiming率 |
| ----- | ------------------ | ------------------ | --------------- |
| 0.0   | 3.1%               | 2.9%               | 31.6%           |
| 0.4   | 7.6%               | 14.1%              | 70.7%           |
| 0.8   | 14.4%              | 32.7%              | 87.1%           |
| 1.0   | 17.6%              | 33.0%              | 87.5%           |

ACDA の claiming 率（≈ wastefulness の代理指標）は本メカニズムの 3〜5 倍に達する。**ACDA は実務で広く使われているにもかかわらず、非常に無駄の多いメカニズムである**ことが定量的に示された。

**学生厚生（平均割当順位、値が小さいほど良い）:**

| alpha | PLDA-RQ | RSDA-RQ | ACDA |
| ----- | ------- | ------- | ---- |
| 0.0   | 1.09    | 1.09    | 1.43 |
| 0.4   | 1.53    | 1.56    | 2.57 |
| 0.8   | 2.66    | 2.83    | 4.19 |
| 1.0   | 3.00    | 3.19    | 4.50 |

alpha が大きいほど学生の選好が相関するため競争が激化し全メカニズムで厚生が下がるが、ACDA は本メカニズムより**常に大幅に劣る**。

### 4.4 なぜ安定マッチングが存在しないのか

最低定員の存在が根本的な問題を引き起こす。

**Example（Example 1 の直感的説明）:**  
学生 $s_1, s_2$ と学校 $c_1, c_2, c_3$ があり、$c_1$ の最低定員が 1（$c_1$ は両学生にとって最不人気）。

- $s_1$ を $c_1$ に割り当てると: $s_2$ は最善の $c_3$ に行けるが、$c_3 \succ_{c_3} s_2 \succ_{c_3} s_1$ なので $s_1$ は $s_2$ に正当な羨望を持つ（fairness 違反）
- $s_2$ を $c_1$ に割り当てると: $s_1$ は最善の $c_2$ に行けるが、同様に $s_2$ が $s_1$ に正当な羨望を持つ（fairness 違反）

どちらの割当でも fairness を満たせない。つまり最低定員がある限り、**必ず誰かが割を食う**構造になっており、それを fairness と nonwastefulness の両方で補償することは不可能である。
